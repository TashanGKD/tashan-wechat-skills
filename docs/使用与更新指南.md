# 使用与更新指南（给智能体）

面向克隆本仓库后要**使用**或**更新**这套 skill 的智能体（Claude Code / Cursor 等）。

---

## 一、它是什么

7 个 skill 组成的公众号发文链路。入口是 `wechat-article-writer`，触发后转发 `document-pipeline`（真正的执行引擎，8 个 Stage，可任意阶段入场），过程中调用 `article-proofreading`（审稿）和 `ai-image-generator`（配图）。

每个 skill **自包含**：`SKILL.md` 是入口，它引用的 `references/`（排版/写作/配图真源）、`assets/`（底部图）、`templates/`（模板）都在同一目录内，路径相对该目录解析。

---

## 二、怎么用

### 2.1 装到 Claude Code

```bash
git clone https://github.com/TashanGKD/tashan-wechat-skills.git
cd tashan-wechat-skills

# 项目级（推荐）：
mkdir -p <你的项目>/.claude/skills
cp -R skills/* <你的项目>/.claude/skills/
# 全局：cp -R skills/* ~/.claude/skills/
# 或： bash scripts/install.sh <你的项目>/.claude/skills
```

**新开一个 Claude Code 会话**（skill 启动时才扫描）。然后：

- 输入 `/wechat-article-writer`，或说「写一篇公众号文章 / 帮我把这篇写完」。
- 已有草稿只想转 HTML：把草稿给它，说「按他山公众号格式排版为 HTML」，它从「已有草稿→引用规范检查→HTML 转换」入场。

### 2.2 装到 Cursor

```bash
cp -R skills/* <你的项目>/.cursor/skills/
```

### 2.3 用软链代替复制（省去每次更新再拷贝）

```bash
for s in skills/*/; do ln -s "$(pwd)/$s" <你的项目>/.claude/skills/$(basename "$s"); done
```

软链后，仓库 `git pull` 更新会自动生效。

## 三、典型产出流程（入口 skill 会带你走）

1. 判断文章类型（科教 / 活动预告 / 活动回顾 / 动态）。
2. 读 `references/写作习惯与风格手册.md`（写作风格）与 `references/微信公众号发布手册.md`（排版规范）。
3. 写稿 / 补全 → 逻辑自检 → 真实性校对。
4. 配图（需 API key，见第五节）→ 图片自审。
5. 审稿（`article-proofreading` 五轮）→ 引用规范检查。
6. 转微信 HTML：品牌绿 `rgb(111,167,170)` 只给 H2/核心 H3/署名标签/协会简介/二维码提示；`strong` 用 `#1a1a1a`；底部模板用 `assets/tashan_footer/`（`qrcode_candidate.png`=山形 logo，`last.jpg`=公众号二维码，记用途不记文件名）。
7. 输出 Markdown 手稿（真源）+ 微信 HTML 到你工作空间的「公众号文章」目录。

## 四、路径约定（最容易踩的坑）

- `SKILL.md` 里的 `references/…`、`assets/…`、`templates/…` **都相对该 skill 自己的目录**。整目录复制/软链，别拆散，否则找不到资源。
- 跨 skill 引用写成 `../<其他skill>/…`（如 `document-pipeline` 引用 `../ai-image-generator/SKILL.md`）。装 skill 时保持它们在同一父目录（`.claude/skills/` 下）即可。

---

## 五、配图需要 API key

`ai-image-generator` 直连阿里云 DashScope：

```bash
export DASHSCOPE_API_KEY=...        # 首选
# 或 AI_GENERATION_API_KEY / LLM_API_KEY
```

- 未设置：只有「生成配图」这步跑不了，写稿 / 审稿 / HTML 转换 / 底部模板不受影响。
- 遇到 401：多半用了后端代理 key（`sk-sp-...`），它不能直连图片端点；换成能直连 DashScope 的 key。
- ⛔ 不要把 key 写进 skill 或提交仓库。

## 六、怎么更新

```bash
cd tashan-wechat-skills
git pull
```

- **复制安装的**：`git pull` 后重装 `bash scripts/install.sh <目标> --force`（或重新 `cp -R`）。
- **软链安装的**：`git pull` 即生效。
- 更新前读 [../CHANGELOG.md](../CHANGELOG.md)：SemVer，**MAJOR = 破坏性变更**，看迁移说明。
- 想改进：按 [../CONTRIBUTING.md](../CONTRIBUTING.md) 开分支 → conventional commit → PR，改动回流本仓库，别只改本地副本。

## 七、排障

| 现象 | 处理 |
|---|---|
| `/wechat-article-writer` 不出现 | 没新开会话；或装错目录（确认在 `.claude/skills/` 或 `.cursor/skills/`）；或该客户端不跟软链 → 改用复制 |
| skill 找不到手册/图片 | skill 目录被拆散了；整目录重装 |
| 配图报 401 / 无法生成 | 见第五节（key 问题） |
| HTML 里绿色过多 | 品牌绿只给 H2/核心 H3/署名/协会简介/二维码提示，`strong` 用 `#1a1a1a` |
