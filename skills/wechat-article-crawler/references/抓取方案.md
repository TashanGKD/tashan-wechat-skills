# 微信公众号文章抓取方案（实战验证）

> 目的：把公开的公众号历史文章归档为本地 Markdown，供研究/复用（本组织自有号，或用户有权访问的公开文章）。
> 核心障碍：`mp.weixin.qq.com` 在非微信客户端下常返回**"环境异常/完成验证"**拦截页——这也是浏览器直接抓、或 WebFetch 常失败的原因。

按可行性排序，选最省力且可行的一条。

---

## 方案一：官方 API（最可靠，需管理员 + 企业认证）✅

前提：是该公众号管理员，后台已开通开发者模式。

```
1. mp.weixin.qq.com → 设置与开发 → 基本配置 → 拿 AppID / AppSecret
2. GET https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=APPID&secret=APPSECRET
3. 用 access_token 调 POST https://api.weixin.qq.com/cgi-bin/freepublish/batchget（已发布文章列表）
```

限制：2025-07 后**个人主体 / 未认证账号**的发布类 API 被取消，仅企业认证账号可用。
⛔ AppSecret 等凭据从环境变量或安全配置读取，**切勿写进 skill / 提交仓库**。

---

## 方案二：网页版 Cookie（已封锁）❌

`wx.qq.com` 扫码登录已被封（"暂不支持网页版微信"）。`wxsid` 是网页版专属 session，手机 App 不产生，无法从手机端代理获取。不要在这条上浪费时间。

---

## 方案三：手机 mitmproxy 抓包（部分可行）⚠️

通过代理可拦到 `pass_ticket` / `wxuin` / `wap_sid2`，但**不足以**调 `profile_ext?action=getmsg`（该接口要 `wxsid`）。

```
1. pip install mitmproxy
2. mitmdump -s wechat_proxy.py --listen-port 8888
3. 手机 Wi-Fi 设代理 → 安装并信任 mitmproxy 证书
4. 手机微信以"普通读者"身份进公众号主页向下滚动，触发 getmsg，脚本捕获 Cookie
```

坑：① iOS 需在「关于本机 → 证书信任设置」额外开启信任；② 管理员访问自己号走管理员接口、不触发 getmsg，须以普通读者身份操作。

---

## 方案四：手动收集链接（最稳）✅

文章不多（< 50 篇）时首选，零对抗、100% 可行。

```
1. 手机微信 → 公众号主页 → 历史消息
2. 逐篇「...」→「复制链接」→ 发到「文件传输助手」
3. 电脑端把链接每行一个存进 links.txt
4. 对每条链接跑 scripts/fetch_wechat.py（见 SKILL.md）
```

---

## 方案五：第三方 RSS 桥（仅近期）⚠️

werss 类服务（如 werss.gqy25.top）把公众号转成 RSS，但：仅近期文章、非全量历史；服务不稳定（实测 2026-03 返 530）。适合"持续订阅新文"，不适合"补全历史"。

---

## 方案六：搜狗微信搜索 ⚠️

`weixin.sogou.com` 可按公众号名搜到近期文章列表，但同样有反爬 + 仅近期，且链接需二次跳转换取真实 URL。作为补充。

---

## 单篇抓取：三策略绕过"环境异常"

`scripts/fetch_wechat.py` 对给定 URL 依次尝试：

1. **移动端微信 UA** + Referer（最轻量，国内 IP 常有效）
2. **PC Chrome UA**（备选）
3. **playwright 无头浏览器**（最可靠，需 `pip install playwright && playwright install chromium`；Windows 上内置 chromium 崩溃时改 `channel="msedge"`）

命中拦截页时脚本显式报错，提示改 `--playwright` 或回落方案四。

⚠️ 易错点：不要把正文里的 `window.__mp_verify_xxx` 误判为拦截——拦截判定只看页面**头部**是否含"环境异常/完成验证"。

---

## 归档输出格式

每篇存为带 YAML frontmatter 的 `.md`：

```markdown
---
title: "..."
source: "他山学科交叉"
source_type: wechat
url: "https://mp.weixin.qq.com/s/xxx"
author: "..."
scraped_at: "2026-..."
---

# 标题

正文…

---
*原文：[来源](url)*
```

去重：以 URL 的 sha256 前缀做 uid，避免重复抓取。

---

## 合规与边界

- 只抓**公开**、且你有权访问的文章（本组织自有号，或公开可读内容），用于归档/研究/风格分析。
- 控制频率，尊重站点；不做大规模商用转载。
- **第三方媒体全文不入公开仓库**（版权）——本 skill 的 `references/他山文章存档/` 只放他山自有文章。研究外部媒体时，本地分析即可，不公开重发。
